Model Assumptions

This simulator uses a simplified constant-product AMM model:

x * y = k amountInWithFee = amountIn * (1 - fee) amountOut = reserveOut * amountInWithFee / (reserveIn + amountInWithFee)

Each swap updates reserves. Output is computed from the fee-adjusted effective input, while the fee remains in the pool, so k can increase after a swap.

This is an educational simulator for EVM-compatible sandwich MEV scenarios. It omits many real-chain execution details such as integer token decimals, routing contracts, builder auctions, mempool latency, and transaction inclusion risk.
