# 2026-04-29 Lab Demo Update

本次更新的目标是让这个 lab 更适合课堂展示三明治攻击，而不只是输出最终利润。

## 更新内容

1. 新增 `trace` 子命令

   可以用下面的命令展示完整攻击过程：

   ```bash
   cd searcher
   cargo run --release -- trace --victim 1000 --slippage 0.01
   ```

   输出会按顺序展示初始池子、攻击者 front-run、victim swap、攻击者 back-run
   四个状态，并列出每一步的输入、输出、储备量和价格。

2. `simulate` 支持固定攻击者规模

   现在可以手动指定攻击者的 front-run 数量：

   ```bash
   cargo run --release -- simulate --victim 1000 --slippage 0.01 --attacker 2000
   ```

   这可以用来展示攻击者投入过大时 victim 交易 revert 的失败案例。

3. 新增攻击者规模扫描

   `sweep` 现在会额外输出：

   ```text
   data/sweep_attacker_size.csv
   ```

   这个 CSV 用来展示攻击者投入规模、攻击者利润、victim 输出和 revert 边界之间的关系。

4. 新增攻击边界图

   Python 绘图脚本现在会生成：

   ```text
   figures/fig_attacker_size.png
   ```

   这张图用于解释为什么最优攻击不是无限 front-run，而是接近 victim 的
   `min_out` 边界。

5. 改进 revert 场景的攻击者损益

   原先 victim revert 时，模拟器把攻击者损失简化成 `-a`。现在会继续模拟攻击者
   unwind，把 front-run 得到的 Y 换回 X，从而更真实地展示失败 sandwich 的亏损。

6. Foundry 测试增加日志

   `forge test -vv` 现在会打印关键链上数值，包括 honest quote、victim
   `minOut`、front-run 输出、victim 实际输出、back-run 输出、攻击者利润和
   victim 额外损失。

7. 新增课堂 walkthrough

   新增文件：

   ```text
   docs/lab_walkthrough.md
   ```

   这个文件按课堂演示顺序组织内容：正常 swap、三步 sandwich、成功/失败对比、
   参数图、链上验证和防御策略。

## 课堂展示效果

这次更新后，lab 可以按下面的顺序演示：

1. 先展示 honest quote 和 slippage `min_out`。
2. 用 `trace` 展示三笔交易如何改变 AMM 状态。
3. 用 `fig_attacker_size.png` 展示最优攻击规模和 revert 边界。
4. 用参数图展示 slippage、流动性深度、手续费和 victim trade size 的影响。
5. 用 Foundry 测试说明这个过程可以在本地 EVM 中复现。
6. 用 defense discussion 收尾，说明如何降低或消除 sandwich 风险。
